# CDP免許更新管理 独立アプリ

このフォルダは、既存のCDP本体Webアプリとは別URL・別スプレッドシートで運用するための独立Apps Scriptです。

既存URL `https://script.google.com/macros/s/AKfycbwxO4B5BKO1gIsJ2FnXOZwn1bUskpcS3FtdySjsK7A/dev` には push / deploy しないでください。

## 運用ルール

- 免許更新は3年に1度
- 更新講習は有効期限の9か月前から受講可能
- 更新申請は有効期間満了日の6か月前から1か月前までに行う
- 更新講習修了証明書は、講習修了日から3か月後の前日までを期限として管理
- 講習修了は更新完了ではない。DIPS申請後、新しい技能証明有効期限を確認して初めて更新完了にする
- 講習後の更新申請フォロー期限は `min(現在の資格有効期限-1か月, 修了証明書有効期限)` で見る
- 次回更新期限は、講習日や入力日から3年後ではなく、DIPSまたは新しい技能証明書に表示された新有効期限で確定する
- ドローン免許証を確認し、有効期限内かを記録する
- 既存受講生と、新規（他講習機関で受けた人）を顧客区分で分ける

## ステータス運用

- 申込済み: 更新講習を申し込んだだけ。更新申請とは別管理。
- 講習修了: 修了証明書が出た状態。DIPS更新申請確認タスクを残す。
- DIPS申請中: 更新完了未確認として管理し、新有効期限確認タスクを残す。
- 更新完了: DIPSまたは新しい技能証明書で更新完了確認日と新有効期限を確認済み。
- 期限超過・未確認: 講習後または申請後も更新完了が確認できない状態。P1アラート対象。

## 初回セットアップ

1. この `renewal_license_app` フォルダを、既存プロジェクトとは別のApps Scriptプロジェクトとして作成します。
2. 作成後、このフォルダ内でだけ `clasp push` / `clasp deploy` を実行します。
3. Webアプリを開き、初回セットアップで専用スプレッドシートを新規作成、または別途用意したスプレッドシートURLを指定します。
4. 設定シートの「内部アラート宛先」に通知先メールアドレスを入れます。
5. 画面の「日次通知設定」で毎日通知を有効にします。

## 読み取り元シート

次のスプレッドシートは読み取り専用のデータ元として扱います。アプリからこのシートへ書き込みは行いません。

https://docs.google.com/spreadsheets/d/1GFMynPtdX1qHCC-GP0rI7XqfWBh_lbC5J_SV91NkN30/edit?gid=307646994#gid=307646994

画面の「読み取り」ボタンを押すと、この元シートから値を読み込み、独立アプリ側の専用管理表 `免許更新管理` に追加・更新します。

読み取り元の見出しは、`氏名`、`免許有効期限`、`有効期限`、`免許番号`、`ステータス`、`更新講習受講日`、`更新講習修了日`、`DIPS申請日`、`新しい技能証明有効期限` などの表記を自動判定します。

## 運用ファイル読込

画面の「運用ファイル読込」ボタンで、担当から共有予定の運用ファイルコピーを読み取り、専用管理表へ反映します。元ファイルは読み取り専用で扱い、書き込みません。

- 更新者一覧: 氏名、メール、有効期限、案内送付状況、講習日、会場、備考を読み取り
- 案内文テンプレート: 開催日程、料金、電話番号、メールアドレスを読み取り
- 講習記録簿: 受講者名、受講日、会場を読み取り、講習修了として反映

読み取り対象URLは設定シートの次の項目で変更できます。

- `運用ファイル_更新者一覧URL`
- `運用ファイル_案内文テンプレートURL`
- `運用ファイル_講習記録簿URL`

講習記録簿から反映しても、資格更新完了にはしません。DIPS申請日、更新完了確認日、新しい技能証明有効期限は別途確認して登録します。

## 推奨機能

- P1/P2だけ日次メール通知
- 個別レコードから内部担当者へ即時メール送信
- 免許証確認日と確認結果を必須管理
- 更新講習申込日、受講日、修了日、更新完了確認日の分離管理
- 修了証明書期限3か月の自動判定
- 更新申請フォロー期限の自動判定
- DIPS申請後の新有効期限確認タスク
- 担当共有の運用ファイルからの自動読込
- 既存/新規（他講習機関）の区分管理
- 次回対応日の手動管理
- アラート送信履歴の自動保存
