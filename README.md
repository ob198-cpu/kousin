# CDP免許更新管理

更新対象者の入力・期限管理と、保存済みレコードから更新講習の成果物を作成する Google Apps Script アプリです。

対象者データの共有正本への自動接続は停止したままです。画面の対象者データはブラウザの `localStorage` に保存され、ユーザーが対象レコードを選び、作成前検査を実行して明示的に確定した場合だけ Drive に成果物を出力します。

## 利用できる成果物作成

- 修了証明書発行台帳
- 更新講習修了証明書
- DIPS提出CSV
- 更新講習案内
- 講習記録簿
- 見積書・請求書

原本テンプレートは変更せず、出力先へコピーまたは新規ファイルを作成します。同じレコード・成果物種別・入力内容で再実行した場合は既存成果物を返し、入力が変わった場合は訂正版として版を進めます。

成果物を作成しただけでは、修了証明書を「交付済」、見積書・請求書を「発行済」、DIPS修了者情報を「連携済」、本人手続を「更新申請済」にはしません。交付、送付、機関連携、本人申請、発行の事実は、実際の業務を確認して別々に記録します。

## 操作の流れ

1. 対象者、航空機の種類、受講開始時の本人・対象講習照合結果を入力し、ブラウザ内台帳へ保存する。
2. データ管理画面で、発行者情報、連絡先、登録番号、振込先、非公開の出力先フォルダ、所有者・編集者・閲覧者・承認済みGoogleグループを含むアクセス許可メール一覧を設定する。出力先は「編集者が権限を変更・共有」を無効にする。
3. 事業者設定で「専用原本を自動準備」を実行する。My Drive直下に制限付きの専用原本フォルダが作られ、実名タブのない修了証明書原本と、既存データのない発行台帳原本のIDが自動設定される。指定された実データ入り参照元IDを直接運用原本にはできない。
4. DIPS担当部署へ5営業日の起算日と閉庁日を確認し、営業日暦の確認日・確認者・追加閉庁日を設定する。未確認の場合、画面は確定期限を表示せず、DIPS CSVの作成前検査を止める。
5. 既存の証明書・DIPS番号を照合し、自動採番の切替年月と必要な最新番号を登録して採番正本への切替を確認する。切替前の講習と照合前は空欄から自動採番しない。
6. 更新講習案内を作る場合は、共有設定へ案内日程を4行登録する。
7. 対象者詳細から「成果物を作成」を開き、必要な成果物を選ぶ。
8. 「作成前検査」で必須項目、日付、時間、料金、設定、テンプレート権限を確認する。
9. 検査合格後に作成を確定し、成果物ごとの作成結果とリンクを確認する。

作成処理は `LockService` で直列化し、作成日時、対象レコードID、種別、入力ハッシュ、テンプレートまたはレイアウト版、版、状態、出力ファイルを生成台帳へ記録します。1種類が失敗しても、ほかの成果物の結果は個別に返します。見積・請求番号は既存CDP管理システムと衝突しない `QT-UC0157-...` / `INV-UC0157-...` を使います。

共有権限の完全照合には、マニフェストで有効化した Advanced Drive service v3 を使用します。Drive APIが無効、権限一覧のページ取得が未完、`domain` / `anyone` 共有、許可外のuser/group、編集者による再共有がある場合は、個人情報を出力しません。許可したGoogleグループのメンバー管理は別途運用監査が必要です。

この版より前に作った `自動作成` フォルダ、対象者別フォルダ、生成台帳、年次発行台帳には固定識別マーカーがないため、自動で自己認定・再利用しません。内容と保存先を管理者が確認し、別名で退避して新規生成するか、承認済みの明示移行を行ってください。既存ファイルの説明欄や見出しを自動上書きして移行済み扱いにはしません。

## 停止中の自動連携

- 対象者データの既存スプレッドシートからの自動読込・自動保存
- 参照用スプレッドシートの自動更新
- Googleドキュメントからの対象者データ自動取込
- メール送信
- DIPSへのログイン、CSVアップロード、申請操作
- 日次通知トリガー
- 対象者データ接続先IDの保存

指定された3つのスプレッドシートは項目対応の参照元とし、そのうち講習記録簿と修了証明書発行台帳だけを成果物テンプレートとして読み取ります。対象者名簿を読み込んだり、テンプレート原本へ直接書き込んだりはしません。案内・講習記録簿も、作成時に想定外タブ、実データ、差込構造の変更を検査して停止します。原本編集者を限定し、変更は承認と試作出力の確認後に反映してください。

ただし、指定された修了証明書原本には実名タブ、発行台帳原本には既存受講者データがあるため、その2ファイル全体は生成原本として使用しません。「専用原本を自動準備」は、証明書を一時コピー上で単一ベース化・固定ダミー値化した後、清浄化済み状態を二次コピーして最終原本にします。一時コピーは完全削除を試み、最終原本の版履歴へ元の個人情報を持ち込みません。台帳はファイル全体をコピーせず、空の`ベース`シートだけを新規ブックへコピーします。両原本の機械検査を通るまで成果物作成を停止します。

DIPS機能は11列のUTF-8 BOM付きCSVを作るところまでです。列7は `PA000000000000`、状態フラグは新規・訂正・削除に対応し、訂正・削除では元の申請者IDを必須にします。CDPが行う「修了者情報連携日」と、受講者本人が行う「DIPS更新申請日」は別項目です。5営業日は前者の期限で、後者の通常期間は現免許期限の6か月前から1か月前までです。公開資料だけでは5営業日の起算日・閉庁日の扱いが確定しないため、翌営業日を1日目とする日付は暫定表示とし、必ず担当部署へ確認します。ログイン後に取得する最新の公式CSVひな形との最終照合も必要です。DIPSの認証情報は保存も使用もしません。

身体適性資料の期限は資料の記載日を入力し、CDPの確認日から自動計算しません。修了証明書の有効期限は初回発行日を基準とし、訂正・再発行日で初回発行日を変更して延長しません。現版は再発行日の別管理に未対応のため、初回正式成果物の作成後に発行日・期限を変更する操作は停止します。

請求・入金画面は、取引日で対象請求を絞り、未発行・取消の参考計算額と、発行済みの債権計上額・実入金額・消込額・未収金・過入金を分けます。未知の税率・端数処理は計算要確認として0円計上にし、経理CSVにも参考計算額と債権計上額を別列で出します。管理年度別の業務集計は会計期間の売上集計ではありません。

## データの位置づけ

`Index.html` の対象者データ、CSV、JSONバックアップ、期限・請求計算はブラウザ内試作です。`localStorage` は端末間で共有されず、権限管理、同時更新、正式な監査、会計締め、講習事務終了後3年間の保存保証を備えないため、共有正本または正式会計帳簿として使用できません。

Driveへ作成した成果物と生成台帳も、元となる対象者データの共有正本を代替しません。正式運用では `OPERATIONS_DESIGN.md` に沿って対象者、案件、証憑、請求、入金、監査ログを分離してください。

## 本番切替前の未完了事項

- 対象者データを共有正本へ移し、複数利用者の権限、同時更新、監査ログ、バックアップ、3年間の保存を実装・試験する
- 個人情報のない証明書・発行台帳原本を承認し、承認済み原本の版またはダイジェストを固定登録する
- 原本と年次台帳の編集範囲を保護し、訂正理由、承認者、承認日時を必須にする
- 実際のDrive権限構成でAdvanced Drive service、作成、再実行、改ざん停止、失敗時復元を結合試験する
- DIPSログイン後の最新公式CSVひな形、5営業日の起算日・閉庁日を担当部署と照合する
- 分割入金、複数請求への一括入金、返金、会計締め、仕訳は正式会計台帳で実装する

これらが完了するまで、本版は成果物の下書き生成・運用検証用であり、本番正本または正式会計帳簿として扱わないでください。

## 実データ連携を広げる場合

成果物作成以外の実データ連携を追加する場合は、接続先、所有者、読取範囲、書込範囲、表示項目、メール送信、権限、バックアップ、デプロイ可否を事前に確認し、別途レビューします。現在許可されているDrive書込は、ユーザーが明示した成果物生成とその生成台帳に限ります。
