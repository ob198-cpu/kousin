# CDP免許更新管理システム 仕様書

作成日: 2026-06-27  
対象: 無人航空機操縦者技能証明の更新講習・更新申請フォロー管理  
方式: 既存CDP本体とは別URL・別スプレッドシートで運用する独立Apps Script

## 1. 目的

無人航空機操縦者技能証明の更新忘れを防ぐため、資格有効期限、更新講習、DIPS更新申請、更新完了確認を一元管理する。

本システムでは、特に次を重視する。

- 「更新講習を受けた日」と「資格更新が完了した日」を別物として管理する。
- 講習修了だけで更新完了扱いにしない。
- DIPSまたは新しい技能証明書で新有効期限を確認して初めて更新完了にする。
- 期限が近い対象者を画面・メールで確認できるようにする。
- 担当から共有される運用ファイルを読み取り、手入力を減らす。

## 2. 前提・制約

- 既存CDP本体Webアプリには干渉しない。
- 既存優先URL `https://script.google.com/macros/s/AKfycbwxO4B5BKO1gIsJ2FnXOZwn1bUskpcS3FtdySjsK7A/dev` へ push / deploy しない。
- 本システムは別Apps Script、別URL、別スプレッドシートで運用する。
- 読み取り元・運用ファイルには書き込まない。
- 元データを読み取った結果は、本システム専用の管理表 `免許更新管理` にのみ反映する。
- DIPSの申請結果は自動取得しない。本人入力、スクリーンショット、代理申請結果などを確認して登録する。

## 3. 対象業務

対象業務は次の通り。

- 更新対象者の一覧管理
- 技能証明有効期限の確認
- 更新講習の案内・申込・日程管理
- 更新講習受講日・修了日の管理
- 修了証明書有効期限の管理
- DIPS更新申請状況の管理
- 更新完了確認日と新しい技能証明有効期限の管理
- 期限超過・未確認者のアラート
- 内部担当者へのメール通知
- 担当共有の運用ファイルからの自動読込

## 5. データ項目仕様

主な管理項目は次の通り。

| 項目 | 内容 |
|---|---|
| 現在の技能証明有効期限 | 更新前または現在有効な技能証明の期限 |
| 更新講習申込日 | スクールへの申込日 |
| 更新講習受講日 | 実際に講習を受けた日 |
| 更新講習修了日 | 修了証明書の発行日 |
| 修了証明書番号 | DIPS申請に使用する番号 |
| 修了証明書有効期限 | 更新講習修了日 + 3か月 - 1日 |
| 更新申請開始可能日 | 現在の技能証明有効期限 - 6か月 |
| 更新申請期限 | 現在の技能証明有効期限 - 1か月 |
| DIPS申請日 | 本人または代理人がDIPS申請した日 |
| 更新完了確認日 | DIPSまたは新しい技能証明書で完了確認した日 |
| 新しい技能証明有効期限 | 更新後に確定した正式な有効期限 |
| 更新申請フォロー期限 | 更新申請期限と修了証明書有効期限の早い方 |

## 6. ステータス仕様

| ステータス | 意味 | システム上の扱い |
|---|---|---|
| 免許証確認待ち | 技能証明書の有効期限確認が未完了 | P1候補 |
| 案内前 | まだ案内していない | 期限に応じてP2/P3 |
| 案内済 | 更新講習案内を送付済み | 申込待ち |
| 申込済み | 更新講習を申し込んだだけ | 更新申請とは別管理 |
| 講習予定 | 講習日が決まっている | 講習実施待ち |
| 講習修了 | 修了証明書が出た状態 | DIPS更新申請確認タスク |
| 更新申請待ち | 講習修了後、DIPS申請待ち | DIPS更新申請確認タスク |
| DIPS申請中 | DIPS申請中 | 新有効期限確認タスク |
| 更新完了未確認 | 更新完了確認が未入力 | P1/P2候補 |
| 更新完了 | 新有効期限と更新完了確認日を確認済み | 次回更新管理へ移行 |
| 期限超過・未確認 | 期限後も更新完了未確認 | P1 |
| 対象外 | 管理対象外 | 通常アラート対象外 |

旧ステータス `講習済・申請待ち` は `講習修了` として扱う。  
旧ステータス `申請完了` は `DIPS申請中` として扱う。

## 7. 期限計算仕様

### 7.1 基本期限

| 項目 | 計算式 |
|---|---|
| 更新講習受講可能日 | 現在の技能証明有効期限 - 9か月 |
| 更新申請開始可能日 | 現在の技能証明有効期限 - 6か月 |
| 更新申請期限 | 現在の技能証明有効期限 - 1か月 |
| 修了証明書有効期限 | 更新講習修了日 + 3か月 - 1日 |
| 更新申請フォロー期限 | min(更新申請期限, 修了証明書有効期限) |

### 7.2 更新完了後

更新完了後の次回更新管理では、`新しい技能証明有効期限` を正式な基準日とする。

禁止する処理:

- 講習完了日から3年後を次回更新期限にする。
- 入力日から3年後を次回更新期限にする。
- 講習修了だけで更新完了にする。

## 8. リスク判定仕様

| リスク | 条件例 | 対応 |
|---|---|---|
| P1 | 免許期限超過、更新完了未確認、更新完了なのに新有効期限未入力 | 即確認 |
| P2 | 申請期限または修了証明期限が30日以内、DIPS申請後の確認期限が近い | 要対応 |
| P3 | 9か月前に入り講習受講可能 | 早期案内 |
| P5 | 期限余裕あり、更新完了、対象外 | 定期確認 |

## 9. メール通知仕様

### 9.1 個別通知

個別レコードから内部担当者へアラートメールを送信できる。

本文に含める主な情報:

- 氏名
- 顧客区分
- 免許番号
- 現在の技能証明有効期限
- 新しい技能証明有効期限
- 状態
- リスク
- タスク
- 理由
- 推奨対応
- 更新講習受講日、修了日
- 修了証明書有効期限
- DIPS申請日
- 更新完了確認日

### 9.2 日次通知

設定シートの `日次通知対象` に設定したリスクを日次メールに含める。初期値は `P1,P2`。

## 10. 読み取り機能仕様

### 10.1 読み取り元シート

画面の `読み取り` ボタンで、指定された読み取り元スプレッドシートを読み込む。

読み取り対象:

- 氏名
- メール
- 免許有効期限
- 免許番号
- ステータス
- 更新講習受講日
- 更新講習修了日
- DIPS申請日
- 新しい技能証明有効期限

読み取り元には書き込まない。

### 10.2 運用ファイル読込

画面の `運用ファイル読込` ボタンで、担当共有予定の運用ファイルを読み込む。

対象ファイル:

| 種別 | 初期URL | 読み取り内容 |
|---|---|---|
| 更新者一覧 | `14bzaYZ_9dz4BMHNVDx7AfpKFN-3pStoWjKdKOMVeiv0` | 氏名、メール、有効期限、案内送付状況、講習日、会場、備考 |
| 案内文テンプレート | `1jmjiJCrmqi_yWNp_hPLfAFmjVctaVqUZDguhRZ-HRks` | 開催日程、料金、電話番号、メールアドレス |
| 講習記録簿 | `1b2gjUL0I2vfK-XOvbDhg8oXg36EdADajamivX7wfgC4` | 受講者名、受講日、会場 |

反映ルール:

- 更新者一覧に講習日がある場合、ステータスを `講習予定` にする。
- 案内送付チェックがある場合、ステータスを `案内済` にする。
- 講習記録簿に受講者名と受講日がある場合、ステータスを `講習修了` にする。
- 講習記録簿反映でも `更新完了` にはしない。

## 11. 設定シート仕様

設定シートで次を変更できる。

- 内部アラート宛先
- 日次アラート時刻
- 更新講習受講可（月前）
- CDP講習・申請受付（月前）
- 手続き停止ライン（月前）
- 更新証明有効（月）
- DIPS申請後確認（日）
- 日次通知対象
- 読み取り元スプレッドシートURL
- 運用ファイル_更新者一覧URL
- 運用ファイル_案内文テンプレートURL
- 運用ファイル_講習記録簿URL

## 12. 権限仕様

Apps Scriptの主な権限:

- スプレッドシート読み書き
- Googleドキュメント読み取り
- Google Drive読み取り
- メール送信
- トリガー作成

読み取り権限は、運用ファイルおよび案内文テンプレートの読込に使用する。

## 13. 非機能要件

- 既存CDP本体へ影響を与えない。
- 元の共有ファイルへ書き込まない。
- 管理表の既存列を壊さないよう、新規項目は後方追加する。
- 日本語画面で運用者が判断できる文言にする。
- 期限計算は画面表示、メール、読込後反映で同じルールを使う。
- 失敗時は画面にエラーまたは注意件数を表示する。

## 14. 受入条件

- Webアプリをブラウザで開ける。
- 初回セットアップで専用管理表を作成できる。
- 手入力で対象者を登録・更新できる。
- 読み取り元シートから対象者を追加・更新できる。
- 運用ファイル読込で更新者一覧・講習記録簿を反映できる。
- 講習修了者が更新完了にならず、DIPS申請確認タスクとして残る。
- 更新完了は新しい技能証明有効期限と更新完了確認日がある場合のみ完了扱いになる。
- P1/P2/P3のリスクが期限に応じて表示される。
- 日次通知と個別通知が送信できる。
- 既存優先URLに影響しない。

## 15. 今後の拡張候補

- 案内文テンプレートから個別メール本文を自動生成する。
- 案内メール送信前チェック一覧を作成する。
- DIPS完了確認用のスクリーンショット提出欄を追加する。
- 更新完了証明の添付ファイル管理を追加する。
- 講習日程別の参加者一覧を自動作成する。
- 監査書類保管チェックを管理表へ連携する。
