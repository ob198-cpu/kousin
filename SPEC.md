# CDP免許更新管理 仕様

## 1. 目的

無人航空機操縦者技能証明の更新漏れを防ぐため、更新期限、講習修了証明書の期限、DIPS申請状況、更新完了確認を分けて管理する。

## 2. 現在の安全モード

現在の実装では、実データへの接続を停止している。

- スプレッドシートを開かない
- Googleドキュメントを開かない
- Driveファイルを開かない
- メールを送信しない
- トリガーを作成しない
- Script Properties に実データIDを保存しない
- 個人名、メール、免許番号、会社名、有効期限を実データから取得しない

## 3. 期限管理ルール

| 項目 | 基準 | システムで見ること |
|---|---|---|
| 現在の資格有効期限 | DIPSまたは技能証明書の有効期限 | すべての基準日 |
| 更新講習開始 | 現有効期限の9か月前 | スクール側の受講案内開始日 |
| DIPS申請開始 | 現有効期限の6か月前 | DIPS申請が可能になる日 |
| 申請期限 | 現有効期限の1か月前 | ここまでに更新申請を進める |
| 講習証明期限 | 講習修了日から3か月後の前日 | 過ぎると再受講リスク |
| 更新完了日 | DIPSまたは新しい技能証明書で確認した日 | スクールの講習日とは別 |
| 新しい資格有効期限 | 更新後のDIPSまたは証明書に表示される期限 | 次回更新管理の正式な基準 |

## 4. アラートルール

更新確認期限は次の早い方を採用する。

```text
更新確認期限 = min(現在の資格有効期限 - 1か月, 講習修了日 + 3か月 - 1日)
```

## 5. 画面に常時出す項目

- 判定
- 対応期限
- 対象
- 必要な対応
- 期限の根拠
- 状態

## 6. 画面に常時出さない項目

- 免許番号
- 電話
- メール
- 会社名
- 修了証明書番号
- 身体適性確認結果
- 内部メモ

これらは対象者詳細画面で必要時だけ表示する。

## 7. 再開時の必須確認

実データ連携を戻す場合は、ユーザーの明示許可を得てから、別ブランチまたは別作業コピーで設計する。

最低限、以下を仕様書に書く。

- 接続先
- 読み取り範囲
- 表示項目
- 書き込み有無
- メール送信有無
- 操作ログ
- デプロイ前レビュー手順
