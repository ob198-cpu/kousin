## 成果物作成で追加する対象者項目

次の項目は既存3シートの列を推測して埋めず、対象者レコードへ明示入力する。空欄をサンプル値で補完しない。

| 画面・CSV項目 | 使用する成果物 | 取扱い |
|---|---|---|
| 技能証明申請者番号 | 講習修了証明書、DIPS提出CSV | 10桁の数字。技能証明書番号とは別項目 |
| 航空機の種類 | 講習修了証明書、講習記録簿 | マルチローター／ヘリコプター／飛行機を明示。証明書区分表の該当1セルだけに印を付ける |
| 受講開始時照合記録 | 発行台帳、講習修了証明書、DIPS提出CSV、講習記録簿 | 申請者番号・氏名・住所・生年月日・DIPS通知講習の一致結果、日付、照合者、証憑参照 |
| 修了証明書の担当講師 | 講習修了証明書 | 記録簿の担当者が全項目で同一の場合だけ入力候補にできる。複数なら手動確認 |
| DIPS申請者ID | DIPS提出CSV | `YYMM`＋月内2桁連番。新規登録だけ空欄なら作成時に採番。訂正・削除は誤登録時と同じID必須 |
| 停止処分者向け講習 | DIPS提出CSV、講習記録簿 | 未確認／なし／あり。正式成果物はなし・ありの確定必須。ありの場合だけ実地講習を必須化 |
| 身体適性検査証明書番号 | DIPS提出CSV | CSVは固定値 `PA000000000000`。画面の番号は内部管理用で転記しない |
| DIPS状態フラグ | DIPS提出CSV | 新規登録／既存情報更新／削除を明示選択 |
| DIPS修了者情報連携日 | DIPS業務管理 | CDPがCSVアップロードの受理を確認した実日。CSV作成日・本人申請日とは別。発行後5営業日の対象 |
| DIPS更新申請日（本人） | 更新案件管理 | 受講者本人が更新申請した実日。通常の6か月前〜1か月前の対象で、機関連携日とは別 |
| 身体適性資料期限 | 更新案件管理 | 使用資料に記載された期限を転記。CDPの確認日から自動計算しない |
| 見積・請求先名 | 見積書・請求書 | 会社名または個人名。対象者名とは別に変更可能 |
| 見積・請求先敬称 | 見積書・請求書 | 御中／様 |
| 見積・請求先住所 | 見積書・請求書 | 請求先として確認した住所 |
| 取引年月日 | 請求書 | 役務提供日。請求日と別に必須 |
| 税率例外承認 | 見積書・請求書 | 0%・8%の場合の承認日、承認者、法令・税務根拠 |
| 見積書番号 | 見積書 | 空欄なら `QT-UC0157-yyyyMMdd-N` を採番。既存番号の再利用は重複検査対象 |
| 見積日 | 見積書 | 空欄時の既定日は作成日。明示入力を優先 |
| 見積有効期限 | 見積書 | 見積日以後であることを検査 |

修了証明書有効期限と証明書番号の年月は、講習修了日ではなく修了証明書初回発行日を基準にする。初回発行日と実際の交付日は別項目として保存し、訂正・再発行日で初回発行日を上書きしない。

```text
修了証明書有効期限 = 修了証明書初回発行日の3暦月後の前日
```

3暦月後に同じ日がない場合は対象月の末日とする（1月31日発行は4月30日まで）。入力済み有効期限が計算値と違う場合は警告し、成果物は計算値で作成する。交付後は、実際に記載した期限を確認記録として残す。

## DIPS提出CSV 11列対応

DIPS提出CSVは次の固定順で作る。UTF-8 BOM付きとし、自動アップロードは行わない。

| CSV列 | DIPS項目 | 画面・生成元 | コード変換 |
|---:|---|---|---|
| 1 | 申請者ID | DIPS申請者ID | `YYMM`＋月内2桁連番 |
| 2 | 技能証明申請者番号 | 技能証明申請者番号 | 10桁 |
| 3 | 登録更新講習機関コード | システム定数 | `0157` |
| 4 | 登録更新講習機関事務所コード | システム定数 | `R0157001` |
| 5 | 区分 | 資格区分 | 一等=`1`、二等=`2` |
| 6 | 停止処分者向け講習受講有無 | 停止処分者向け講習 | なし=`1`、あり=`2` |
| 7 | 身体適性検査証明書番号 | システム定数 | `PA000000000000` |
| 8 | 更新講習修了証明書番号 | 更新講習修了証明書発行番号 | `UC0157YYMMNNNN` |
| 9 | 更新講習修了日 | 講習修了日 | `yyyy/MM/dd` |
| 10 | 有効期間満了日 | 修了証明書有効期限 | `yyyy/MM/dd` |
| 11 | 状態フラグ | DIPS状態フラグ | 新規登録=`1`、既存情報更新=`2`、削除=`3` |

この列対応は公開中の国土交通省ガイドラインとFAQへ合わせている。DIPSへログインしてダウンロードする最新CSVひな形との最終照合は未実施のため、使用前に担当者が列名、順序、コード値を照合し、確認日と確認者を残す。

CDP受講分だけをCDPのCSV作成対象とする。修了者情報連携は修了証明書発行後5営業日以内とされるが、公開資料だけでは起算日・閉庁日の扱いを確定できない。翌営業日を1日目とする値は暫定であり、内閣府祝日、追加閉庁日、担当部署への確認日・確認者が揃わなければ確定期限を表示しない。

## 更新講習案内の日程マスタ

日程マスタは4行とし、各行に次を保存する。

| 項目 | 取扱い |
|---|---|
| 日付 | `YYYY-MM-DD`。対象者の講習実施可能日から講習締切日の範囲内 |
| 会場 | 案内へ表示する会場名 |
| 午前・午後・夜間 | 各行で1つ以上の受付可能時間帯を選択 |

日程マスタはScript Propertiesの共有設定へ保存し、ブラウザの `localStorage` は読込表示用キャッシュだけにする。見本日付、見本会場、見本料金を対象者データへ自動転記せず、案内作成前に実際の開催予定を確認する。

## 見積・請求項目の計算対応

| 入力項目 | 帳票項目 | 計算 |
|---|---|---|
| 料金（税抜） | 税抜料金 | 正の金額 |
| 値引（税抜） | 税抜値引 | 絶対額として差し引く |
| 税率 | 税率・消費税 | 0%、8%、10%を区分して扱う |
| 端数処理 | 消費税 | 切捨て、四捨五入、切上げ |

```text
税抜対象額 = max(0, 料金税抜 - 値引税抜)
消費税     = 端数処理(税抜対象額 × 税率)
税込合計   = 税抜対象額 + 消費税
```

複数明細へ拡張する場合は税率別税抜合計を作り、適格請求書1件につき税率ごとに消費税を1回だけ丸める。
